# Standards Index

The standards ecosystem for space software — what each one covers and when it applies.

---

## Software Development Standards

| Standard | Body | Domain | Key Scope |
|---|---|---|---|
| **DO-178C** | RTCA | Avionics (also used in space) | Software development and verification process, DAL A–E |
| **ECSS-E-ST-40** | ESA | Space | Software engineering process for ESA missions |
| **NPR 7150.2** | NASA | Space | NASA software requirements — Class A–D classification |
| **MIL-STD-498** | DoD | Defense | Defense software development (largely superseded by IEEE 12207) |
| **IEEE 12207** | IEEE | General | Software lifecycle processes |

## Relationship Between Standards

```
DO-178C ←── Used by NASA missions voluntarily (especially for DAL A/B)
     ↑
     │
ECSS-E-ST-40 ←── ESA-funded missions (Sentinel, BepiColombo, JUICE)
     ↑
     │
NPR 7150.2 ←── All NASA programs (Class A = highest, D = lowest rigor)
```

For commercial small satellites, most companies apply MISRA-C and internal coding standards without formal DO-178C certification. Certification becomes required when the spacecraft is safety-of-life (crew, life support) or when selling to a customer that requires it (e.g., FAA-regulated launch).
