# Certification Path

Certification is the formal process of demonstrating to a regulatory authority (FAA, ESA, NASA, DOD) that your software is safe for its intended use. The path depends on the mission type, safety criticality, and the regulatory framework that applies.

---

## Which Standard Applies?

| Mission Type | Regulatory Body | Standard |
|-------------|----------------|---------|
| Civil aviation (incl. launch vehicles) | FAA | DO-178C |
| ESA scientific / commercial | ESA | ECSS-E-ST-40C |
| NASA human spaceflight | NASA | NPR 7150.2 / DO-178C |
| NASA robotic | NASA | NPR 7150.2 (DAL varies) |
| DoD military space | USAF/USSF | MIL-STD-498, DO-178C |
| Commercial launch (US) | FAA AST | FAA Order 8110.49 |
| Commercial satellite (US) | FCC (comms only) | FCC rules |

## DO-178C Certification Path

### Phase 1: Planning

Produce and gain authority approval for:

- **PSAC** (Plan for Software Aspects of Certification)
- **SDP** (Software Development Plan)
- **SVP** (Software Verification Plan)
- **SCMP** (Software Configuration Management Plan)
- **SQAP** (Software Quality Assurance Plan)

Authority reviews and issues a **Stage of Involvement (SOI) 1** approval.

### Phase 2: Development

Develop software to the plans. Key milestones:

| Gate | Deliverable | Review |
|------|-------------|--------|
| SRR | Software Requirements | Authority SOI-1 |
| PDR | High-level design | Internal |
| CDR | Low-level design + code | Authority SOI-2 |
| TRR | Completed tests | Authority SOI-3 |
| FAI | Final article inspection | Authority SOI-4 |

### Phase 3: Verification

Produce evidence that all objectives are met:

```
Software Verification Results (SVR):
  ├── Test reports (all tests pass)
  ├── Coverage reports (MC/DC 100%)
  ├── Static analysis reports (all findings resolved)
  ├── Review records (all items dispositioned)
  └── Traceability matrices (all requirements covered)
```

### Phase 4: Software Accomplishment Summary (SAS)

The SAS is the top-level certification document. It states:

- Software level and functions
- Life cycle processes used
- Deviations from the standard
- Evidence that all objectives are met
- Tool qualification status

The authority reviews the SAS and grants approval.

## ECSS Certification Path

ESA uses a **Product Assurance (PA) Review** gate structure:

| Gate | Name | Content |
|------|------|---------|
| MDR | Mission Definition Review | Feasibility |
| SRR | System Requirements Review | System spec |
| PDR | Preliminary Design Review | Design + FMEA |
| CDR | Critical Design Review | Detailed design + code |
| QR | Qualification Review | Test results |
| AR | Acceptance Review | Flight acceptance |

Software deliverables at each gate are defined in the **PA Plan** and documented in **DRDs (Document Requirements Definitions)**.

---

## Tailoring

Both DO-178C and ECSS allow tailoring — formally documenting and justifying why certain requirements do not apply or are satisfied by alternative means:

```
Deviation Record: DR-SW-042
  Requirement: ECSS-E-ST-40C, Section 6.4.2
                "Code shall be compiled with -O0"
  Justification: Mission requires -O2 for timing constraints.
                 Equivalence verified by comparing assembly output
                 and re-running all coverage tests with -O2.
  Authority approval: ESA TEC-SWE, 2024-02-01
```

Every deviation requires documented rationale and authority approval.

## Evidence Package

A complete software certification evidence package typically contains:

| Document | Volume (DAL A) |
|---------|---------------|
| Plans (5 plans) | 50–200 pages each |
| Requirements (HLR + LLR) | Hundreds to thousands |
| Design documentation | 200–500 pages |
| Code review records | 1 per file/module |
| Static analysis reports | Exported from tool |
| Test specifications | 1 per test case |
| Test results | Automated logs |
| Coverage reports | Generated by tool |
| SAS | 50–100 pages |
| Total | **Thousands of pages** |

Budget significant schedule for evidence production — it is often the longest phase.

---

## Commercial Space Fast-Path

For commercial missions not requiring FAA Part 450 or DOD approval:

1. Internal safety review board
2. Range safety review (for launch)
3. FCC license (for comms)

No DO-178C required — but many companies voluntarily apply it for rigor, insurance reasons, or government customer requirements.
