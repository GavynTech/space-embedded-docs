Triple Modular Redundancy Voter

Triple Modular Redundancy (TMR) is the oldest and most reliable fault-tolerance technique in safety-critical systems. Run the same computation three times independently, compare the results, and output the majority.

A single fault — hardware or software — can only affect one of the three copies. The other two agree, and the voter outputs the correct value.

The Principle​

Input → Compute A ──┐
Input → Compute B ──┼──→ Voter → Output (majority)
Input → Compute C ──┘

If A=1, B=1, C=0 → output 1 (A and B agree)
If A=1, B=0, C=0 → output 0 (B and C agree)
If A=1, B=1, C=1 → output 1 (all agree)

TMR tolerates any single fault. It does not tolerate two simultaneous faults affecting different channels — for that you need quadruple redundancy or other techniques.

Integer Voter​

#include <stdint.h>
#include <stdbool.h>

typedef enum {
    VOTE_RESULT_UNANIMOUS,  // all three agree
    VOTE_RESULT_MAJORITY,   // two agree, one disagrees (fault detected)
    VOTE_RESULT_FAILED,     // all three disagree (cannot determine correct value)
} VoteResult_t;

// Vote on three 32-bit integer values
// Returns the majority value, result indicates agreement level
uint32_t tmr_vote_u32(uint32_t a, uint32_t b, uint32_t c,
                      VoteResult_t* result) {
    if (a == b && b == c) {
        *result = VOTE_RESULT_UNANIMOUS;
        return a;
    } else if (a == b) {
        *result = VOTE_RESULT_MAJORITY;  // c is the outlier
        return a;
    } else if (a == c) {
        *result = VOTE_RESULT_MAJORITY;  // b is the outlier
        return a;
    } else if (b == c) {
        *result = VOTE_RESULT_MAJORITY;  // a is the outlier
        return b;
    } else {
        *result = VOTE_RESULT_FAILED;
        return a;  // arbitrary — caller must handle FAILED case
    }
}

Floating-Point Voter with Tolerance​

For floating-point values (sensor readings, computed states), exact equality is unreliable due to rounding. Use a tolerance-based voter:

#include <math.h>
#include <float.h>

// Vote on three double values within a tolerance
// Returns the median (robust to one outlier) and reports divergence
double tmr_vote_double(double a, double b, double c,
                       double tolerance,
                       VoteResult_t* result) {
    double diff_ab = fabs(a - b);
    double diff_ac = fabs(a - c);
    double diff_bc = fabs(b - c);

    bool ab_agree = (diff_ab <= tolerance);
    bool ac_agree = (diff_ac <= tolerance);
    bool bc_agree = (diff_bc <= tolerance);

    if (ab_agree && ac_agree && bc_agree) {
        *result = VOTE_RESULT_UNANIMOUS;
        return (a + b + c) / 3.0;   // average for minimum noise
    } else if (ab_agree) {
        *result = VOTE_RESULT_MAJORITY;   // c is outlier
        return (a + b) / 2.0;
    } else if (ac_agree) {
        *result = VOTE_RESULT_MAJORITY;   // b is outlier
        return (a + c) / 2.0;
    } else if (bc_agree) {
        *result = VOTE_RESULT_MAJORITY;   // a is outlier
        return (b + c) / 2.0;
    } else {
        *result = VOTE_RESULT_FAILED;
        // All three disagree — return median as best guess
        double sorted[3] = {a, b, c};
        // Simple 3-element sort
        if (sorted[0] > sorted[1]) { double t=sorted[0]; sorted[0]=sorted[1]; sorted[1]=t; }
        if (sorted[1] > sorted[2]) { double t=sorted[1]; sorted[1]=sorted[2]; sorted[2]=t; }
        if (sorted[0] > sorted[1]) { double t=sorted[0]; sorted[0]=sorted[1]; sorted[1]=t; }
        return sorted[1];   // median
    }
}

TMR-Protected Critical Variable​

For variables so critical that a single SEU must not corrupt them — spacecraft mode, thruster enable flags, communication encryption keys:

// TMR-protected variable — stored in three separate memory locations
// Linker can be configured to place these in different SRAM banks
typedef struct {
    uint32_t copy_a;
    uint32_t padding_a[15];  // pad to separate cache lines / memory regions
    uint32_t copy_b;
    uint32_t padding_b[15];
    uint32_t copy_c;
    uint32_t padding_c[15];
} TMR_Uint32_t;

void tmr_write(TMR_Uint32_t* tmr, uint32_t value) {
    tmr->copy_a = value;
    tmr->copy_b = value;
    tmr->copy_c = value;
}

uint32_t tmr_read(TMR_Uint32_t* tmr, bool* fault_detected) {
    VoteResult_t result;
    uint32_t value = tmr_vote_u32(tmr->copy_a, tmr->copy_b, tmr->copy_c, &result);

    if (result == VOTE_RESULT_MAJORITY) {
        // One copy was corrupted — correct all three copies
        tmr_write(tmr, value);
        *fault_detected = true;
        log_event(EVENT_TMR_CORRECTION, (uint32_t)tmr, 0);
    } else if (result == VOTE_RESULT_FAILED) {
        // Severe corruption — all three disagree
        // Best we can do: use majority of bits across the three values
        *fault_detected = true;
        fdir_report_anomaly(ANOMALY_TMR_FAILURE, (uint32_t)tmr);
    } else {
        *fault_detected = false;
    }

    return value;
}

// Usage: spacecraft mode register
static TMR_Uint32_t g_spacecraft_mode;

void set_spacecraft_mode(SpacecraftMode_t mode) {
    tmr_write(&g_spacecraft_mode, (uint32_t)mode);
}

SpacecraftMode_t get_spacecraft_mode(void) {
    bool fault;
    uint32_t mode = tmr_read(&g_spacecraft_mode, &fault);
    return (SpacecraftMode_t)mode;
}

TMR for Complete Function Execution​

On processors with lockstep support (ARM Cortex-R with lockstep, LEON3 with FT mode), TMR can be applied at the hardware level — two or three CPU cores execute the same instructions and results are compared cycle by cycle. A discrepancy triggers a fault.

In software, you can implement TMR for function outputs:

// Run a critical function three times and vote on the result
// Used for computations where a transient SEU during execution could corrupt output

double compute_control_torque_tmr(const ControlInput_t* input) {
    // Run three independent computations
    double result_a = compute_control_torque(input);
    double result_b = compute_control_torque(input);
    double result_c = compute_control_torque(input);

    VoteResult_t vote;
    double voted = tmr_vote_double(result_a, result_b, result_c,
                                   TORQUE_TOLERANCE_NM, &vote);

    if (vote != VOTE_RESULT_UNANIMOUS) {
        log_event(EVENT_TMR_SOFT_FAULT, 0, 0);
    }

    return voted;
}

This is expensive — 3x compute cost — and only justified for functions where a corrupted output has catastrophic consequences (thruster firing commands, safe mode transitions).
